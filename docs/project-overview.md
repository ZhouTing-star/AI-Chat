# AI Chat Platform 项目梳理

## 1. 项目概述

AI Chat Platform 是一个前后端分离的智能对话系统，核心目标是把「通用 AI 对话能力」和「本地知识库问答能力」做成一个完整闭环。

系统围绕两条主链路展开：

1. 对话链路：用户发问后，前端调用流式接口，后端转发上游大模型结果并实时回推，前端按增量内容逐字渲染。
2. 知识链路：用户上传文档后，后端完成文本解析、分块、索引与检索，检索结果再注入对话上下文，最终生成带引用的回答。

在产品形态上，项目不仅支持多会话聊天，还支持会话级附件注入、知识库管理、检索测试、引用可视化与模式切换，能够覆盖“自由问答 + 文档问答”两类场景。

## 2. 技术栈与选型

### 2.1 前端

1. React 19 + TypeScript + Vite：保证开发体验、类型安全与构建效率。
2. Zustand + persist：按领域拆分状态仓库并持久化，降低组件耦合。
3. Tailwind CSS 4：快速构建响应式界面。
4. react-window：处理长消息列表虚拟滚动。
5. react-markdown + remark-gfm + react-syntax-highlighter：实现 AI 回答富文本展示。
6. pdfjs-dist + JSZip：前端附件解析能力，支持对话侧直接补充上下文。

### 2.2 后端

1. Node.js + Express 5：提供聊天、知识库、上传与检索 API。
2. axios + eventsource-parser：对接上游模型并解析流式返回。
3. multer：多类型文档上传。
4. pdf-parse + mammoth：PDF、DOCX 文档解析。
5. better-sqlite3：本地数据存储与检索索引。

### 2.3 检索层

1. SQLite FTS5：关键词倒排检索。
2. 向量检索：Embedding + 余弦相似度评分。
3. 混合检索：向量分与关键词分融合排序，支持严格/平衡/通用策略切换。

## 3. 核心功能实现

### 3.1 基于 SSE 的实时 AI 对话

1. 前端通过 fetch 发起 text/event-stream 请求，建立持久连接。
2. 自定义解析器按事件边界处理 message、done、citations、error。
3. 通过 chunk 增量追加实现打字机效果，确保响应过程可见。
4. 前端实现暂停、继续、重生，后端实现异常回包与连接收尾。

### 3.2 基于 Zustand 的分层状态管理

1. 将状态按职责拆分为 session、chat、ui、theme、user 等独立 store。
2. 使用 Map 结构存储 messagesBySession，实现多会话快速定位。
3. 结合 persist 中间件完成本地恢复，保持刷新后会话连续性。
4. 采用不可变更新方式，降低共享状态引发的副作用风险。

### 3.3 高性能聊天与滚动体验

1. 使用 react-window 仅渲染可视区消息，降低长对话 DOM 压力。
2. 结合 ResizeObserver 动态测量消息高度，避免高度估算误差。
3. 使用 debounce 控制高频滚动到底部操作，减少抖动与重排。
4. 消息组件使用 memo，图片使用 lazy loading，减少无效渲染开销。

### 3.4 多会话管理与流式增量更新

1. 支持新建、切换、重命名、删除会话，会话与消息数据互相隔离。
2. 为每个会话维护独立流式任务与关闭句柄，避免串流污染。
3. 在流式阶段通过 addChunkMessage 对最后一条 AI 消息做增量拼接。
4. 在删除会话时同步清理附件和消息，保持状态一致性。

### 3.5 本地知识库管理与检索验证

1. 支持新建知识库、启停知识库、上传文档、启停文档、删除文档。
2. 提供重建索引与测试检索能力，可快速验证知识命中质量。
3. 系统状态面板展示真实存储、已激活文档数与索引状态。
4. 回答中展示引用来源与相似度，提升结果可追溯性。

### 3.6 文档入库与混合检索

1. 对 PDF、DOCX、TXT/MD/CSV/JSON 文档进行解析并统一分块。
2. 分块结果写入 chunks 与 FTS 表，同时尝试生成向量。
3. 向量不可用时允许降级入库，后续可通过重建索引补全。
4. 检索时支持关键词与向量混合排序，并根据模式控制阈值。

### 3.7 通用能力沉淀

1. ErrorBoundary 负责运行时异常兜底，防止单点错误拖垮全页。
2. WithPermission 统一权限控制逻辑，降低散落在页面中的条件判断。
3. 服务层封装 chatStream 与 knowledgeApi，减少业务组件直接耦合网络细节。

## 4. 技术亮点

1. 全链路闭环：实现了从上传、解析、索引、检索、注入到引用展示的完整 RAG 流程。
2. 实时交互体验：SSE 增量渲染 + 可中断可恢复机制，兼顾流畅性与可控性。
3. 可维护状态架构：多 store 分层 + 持久化 + Map 结构，支持复杂会话场景演进。
4. 性能治理明确：虚拟滚动、动态测量、去抖与组件记忆化共同保障长对话性能。
5. 工程容错完善：解析失败降级、向量失败兜底、接口错误透传与前端可感知提示。
