# 简历项目写法（AI Chat Platform）

以下内容可直接用于简历项目经历，也可按投递岗位做裁剪。

## 1. 标准版（推荐）

项目名称：AI Chat Platform（智能对话与本地知识库增强）

项目描述：
基于 React19 + Vite + Node.js 构建前后端分离的智能对话系统，完成多会话管理、SSE 流式回答、知识库文档解析入库、RAG 检索增强与引用透明展示。系统支持严格/平衡/通用三种回答模式，并实现会话级附件解析与隔离，适用于智能助手、学习问答、文档问答场景。

主要技术栈：
React 19 + TypeScript + Vite + Zustand + Tailwind CSS + react-window + Node.js + Express + SSE + SQLite(FTS5) + better-sqlite3 + pdf-parse + mammoth

技术亮点：
1. 基于 SSE 实现流式对话链路，完成 chunk 增量渲染、暂停继续、重新生成与异常回退，提升实时交互体验。
2. 设计严格/平衡/通用三档回答模式：严格模式无命中拒答，通用模式跳过检索降压，平衡模式兼顾知识命中与可用性。
3. 构建本地 RAG 检索能力：完成文档上传解析、分块入库、FTS 关键词检索、向量检索与混合排序。
4. 实现会话级附件能力：附件按 session 隔离、上传中禁发、会话删除联动清理，避免上下文串扰。
5. 完成 AI 消息富文本渲染与引用透明度展示，支持 Markdown、代码高亮、Top-K 引用展开与低质量引用折叠。
6. 通过 react-window + 动态行高测量优化长对话性能，并实现“用户上滑查看历史时自动取消跟随”。


## 2. 校招精简版（3-4 条）

项目名称：AI Chat Platform

项目描述：
搭建智能对话系统，支持多会话、流式回答、知识库检索增强与引用追溯，覆盖文档问答核心链路。

技术亮点：
1. 负责 SSE 流式对话前后端实现，支持暂停/继续/重生与错误兜底。
2. 负责 RAG 检索能力建设，完成文档解析、切片、FTS+向量混合检索与模式策略切换。
3. 负责会话状态与附件状态治理，实现会话隔离、上传中禁发与历史恢复。
4. 引入虚拟滚动与滚动策略优化，保障长对话场景的渲染与交互性能。


## 3. 后端偏向版

项目名称：AI Chat Platform（后端核心）

项目描述：
负责 Node.js/Express 后端能力建设，提供模型流式转发接口、知识库管理接口与 RAG 检索服务。

技术亮点：
1. 设计并实现 `/api/chat/stream`，完成上游模型 SSE 转发、上下文裁剪、异常处理与保活机制。
2. 实现文档上传解析链路，支持 PDF/DOCX/文本文件入库，完成切片与索引构建。
3. 实现 SQLite FTS5 + 向量混合检索，支持多模式阈值策略与文档范围收敛。
4. 通过配置化模型接口参数实现多厂商适配，降低模型切换成本。


## 4. 前端偏向版

项目名称：AI Chat Platform（前端核心）

项目描述：
负责 React 前端架构与核心交互，完成会话管理、流式渲染、模式切换、附件解析与性能优化。

技术亮点：
1. 封装流式渲染状态机，支持 chunk 增量显示、暂停/继续/重生与错误提示。
2. 基于 Zustand 构建会话、消息、UI、主题分仓状态，落地本地持久化与恢复。
3. 实现会话级附件解析（pdfjs + JSZip）与隔离策略，提升文档问答可用性。
4. 基于 react-window 实现虚拟列表并优化自动跟随策略，显著提升长列表交互体验。


## 5. 可量化表达模板（可替换）

1. 在百条以上消息场景下，通过虚拟滚动将页面渲染卡顿明显降低。
2. 通过回答模式与检索策略优化，减少无效检索请求并提升回答可控性。
3. 完成文档问答闭环后，缩短用户从上传到可问答的操作链路。
